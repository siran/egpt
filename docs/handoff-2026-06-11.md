## Latency reality (for the operator, already explained)
Transport (LAN/HTTP/Beeper) is ~ms — it can NOT make a brain faster. Slowness = model +
context size. CLI "feels" fast because it STREAMS tokens; Beeper has no clean message-edit, so
replies arrive as one blob (dead-air then text). The real levers over WhatsApp: smaller model
(haiku / local llama) + smaller context (renew/compact) + warm + dispatch order — all done
except compaction policy (#3). Streaming-feel is not achievable on this transport.
