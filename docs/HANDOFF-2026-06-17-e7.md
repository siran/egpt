## Landmines (learned the hard way this session)
- **Work in a separate worktree for multi-edit changes** — `/restart` respawns from
  the MAIN checkout's working tree, so an uncommitted half-edit on main can go live.
- **node_modules in a throwaway worktree: `npm install` it, don't junction.** A
  junction + `Remove-Item -Recurse` followed the link into MAIN's node_modules and
  deleted `vitest` (recovered with `npm install`). If you must junction, remove the
  link with `cmd //c rmdir` and VERIFY before deleting the parent.
- **The headless-runtime shim renders synchronously** — mount effects (e.g.
  `engine.setSubmit`, `engine.configureGate`) run during `render(h(App))`, before the
  boot's `await engine.startAttach()`. Don't rely on async-Ink timing.
- **The deploy poll for a new pid can read `alive.txt` mid-write** (gives a false
  "same pid"); confirm the respawn via `nucleus.json` + `egpt.log`, not the poll alone.
- **Flaky test (unidentified):** the full suite occasionally reports ~9 fewer +
  "1 skipped", green on every re-run. Pre-existing, not from this session's changes —
  worth hunting once Phase C lands (likely a timing/resource-probed dynamic suite).
